# TraceMind AI No-Voice Demo Guide

This version is designed for a `46-50 second` screen recording with `no voiceover`. Instead of narration, the demo relies on smooth cursor movement and the built-in hover cues now added to the main UI sections.

## Recording Goal

Show that TraceMind AI:

- turns one requirement into structured QA outputs
- presents the workflow clearly
- includes persistence, traceability, and testing thinking
- looks polished enough for recruiter and manager review

## 46-50 Second Sequence

### 0:00-0:06

Open on the hero section. Move the mouse over the workflow snapshot on the right and pause long enough for the `Overview cue` popup to appear.

Viewer takeaway:
`This project takes a requirement through a clear workflow from input to structured outputs.`

### 0:06-0:11

Scroll slightly to the output coverage area. Hover the output grid and pause for the `Coverage cue` popup.

Viewer takeaway:
`The app is organized around multiple output types, not a single block of generated text.`

### 0:11-0:19

Move to the workflow generator. Hover the panel for the `Input cue`, then click `Use Sample Requirement`.

Viewer takeaway:
`One requirement, domain context, and risk level drive the whole review package.`

### 0:19-0:24

Click `Generate Demo Package`. Let the loading state show briefly, then allow the page to scroll into the results.

Viewer takeaway:
`The app actively generates a structured review package instead of just showing static content.`

### 0:24-0:31

Hover the top of the results section for the `Results cue`. Then move into `Primary Workflow Paths` and pause for the `Workflow cue`.

Viewer takeaway:
`The dashboard converts one requirement into organized workflow tests and QA artifacts.`

### 0:31-0:37

Scroll slightly and hover `Audit and Traceability Matrix` for the `Traceability cue`.

Viewer takeaway:
`The project shows requirement-to-verification mapping, which makes it feel more credible and product-minded.`

### 0:37-0:43

Hover `Risk Notes and Guardrails` for the `Risk cue`.

Viewer takeaway:
`The project demonstrates QA judgment, not just UI or automation output.`

### 0:43-0:48

Scroll to `Saved Run History` and hover for the `History cue`. If there is a saved item visible, click it once to reinforce persistence.

Viewer takeaway:
`The app supports saved analyses and repeatable review flow.`

### 0:48-0:50

End on the results header with the export buttons visible, or hold on the saved-history area if it looks cleaner in the recording.

Viewer takeaway:
`TraceMind AI is a polished, review-ready portfolio project.`

## Cursor Style Tips

- Move slowly and deliberately.
- Pause `2-3 seconds` on each hover cue.
- Avoid fast scrolling.
- Keep the mouse near the highlighted panels instead of circling the whole screen.
- If a popup appears too briefly, hover again rather than rushing forward.

## Best Recording Setup

- Use desktop view, not mobile view.
- Keep browser zoom around `100%` or `110%`.
- Close unrelated tabs and windows.
- Make sure the backend is running if you want saved history and live generation to appear.
- Record at `1080p` if possible.
