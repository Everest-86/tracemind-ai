# TraceMind AI Feedback Survey

Use this survey after someone reviews the repository, watches the short demo, or tries the local app. The goal is to collect feedback that improves clarity, professionalism, and portfolio impact without overcomplicating the project.

## Best Use

- Ask the reviewer to spend 2-5 minutes with the README first.
- If possible, also show the app or screen recording.
- Keep the survey short enough that someone can finish it in under 3 minutes.
- Use the general survey below for most reviewers.
- Use the audience-specific versions in `docs/` when you want more targeted feedback.

## General Survey

### 1. What best describes your role?

Choose one:

- Hiring Manager
- Engineering Manager
- Software Engineer
- QA Engineer / Test Engineer
- Product Manager
- Designer / UX Reviewer
- Founder / Client / Consultant
- Student / Peer Reviewer
- Other

### 2. After reviewing TraceMind AI, how clear was the project purpose?

Response type: `1-5 rating`

- 1 = unclear
- 5 = very clear

### 3. How professional does the GitHub repository look overall?

Response type: `1-5 rating`

### 4. How easy was it to understand the project structure and architecture?

Response type: `1-5 rating`

### 5. How clearly does the README explain what the project does?

Response type: `1-5 rating`

### 6. How well does the project demonstrate AI-assisted workflow automation in a practical way?

Response type: `1-5 rating`

### 7. How strongly does the project communicate QA, testing, and documentation thinking?

Response type: `1-5 rating`

### 8. Does the project feel like a strong portfolio piece for technical hiring review?

Response type: `1-5 rating`

### 9. What part of the project felt strongest?

Response type: `short answer`

### 10. What part was unclear, weak, or less convincing?

Response type: `short answer`

### 11. What should be improved first to make this project more impressive?

Response type: `short answer`

### 12. What skills does this project communicate to you?

Response type: `short answer`

### 13. After seeing this project, how interested would you be in learning more about the candidate?

Choose one:

- Very interested
- Somewhat interested
- Neutral
- Less interested

### 14. Any final suggestions?

Response type: `paragraph`

## Recommended Short Version

If you want a faster survey, use these 8 questions only:

- What best describes your role?
- How clear was the project purpose?
- How professional does the repository look?
- How easy was the architecture and structure to understand?
- How strongly does the project show QA and testing thinking?
- What part felt strongest?
- What should be improved first?
- How interested would you be in learning more about the candidate?
